เอกสารนี้อธิบายว่าไฟล์แต่ละตัวทำหน้าที่อะไร, ที่เก็บรหัสลับ (.env) อยู่ตรงไหน, คำสั่ง Git ที่ใช้บ่อย, และวิธีรันโปรเจกต์ — เขียนสำหรับคนที่ไม่มีพื้นฐานคอมพิวเตอร์มาก่อน

01ภาพรวมระบบทำงานยังไง

ข้อมูลอากาศเดินทางจากเซนเซอร์จริงในฟาร์ม มาจนถึงหน้าจอที่คุณเห็น ผ่าน 5 ขั้นตอนนี้

เซนเซอร์ AETHER1
วัดค่าอากาศจริงที่ฟาร์มสระบุรี
Node-RED
รันบน Jetson Nano รับค่าแล้วส่งต่อ
backend (FastAPI)
โฟลเดอร์ backend/ — รับ ตรวจสอบ บันทึกข้อมูล
Supabase
ฐานข้อมูลเก็บค่าอากาศ + ผลทำนาย
เว็บ (frontend)
โฟลเดอร์ frontend/ — หน้าจอที่คุณเห็น
เข้าใจง่ายๆ โปรเจกต์นี้แบ่งเป็น 2 ฝั่งเสมอ: backend/ คือ "สมอง" ที่รับ-เก็บ-คำนวณข้อมูลอยู่เบื้องหลัง ส่วน frontend/ คือ "หน้าตา" เว็บที่คนเปิดดู ทั้งสองฝั่งคุยกันผ่านอินเทอร์เน็ต ไม่ใช่ไฟล์เดียวกัน

02โครงสร้างโฟลเดอร์

แผนผังไฟล์ทั้งหมดในโปรเจกต์ — ไม่ต้องจำทุกบรรทัด แค่รู้ว่าอะไรอยู่โซนไหน

voice-ai/ ├── README.md คำแนะนำเริ่มต้น (เปิดอ่านก่อนอันอื่น) ├── .gitignore บอก Git ว่าไฟล์ไหนไม่ต้องอัปโหลด (เช่น .env) │ ├── backend/ ← "สมอง" ของระบบ (Python) │ ├── main.py โค้ดเซิร์ฟเวอร์ทั้งหมดอยู่ที่นี่ไฟล์เดียว │ ├── requirements.txt รายชื่อไลบรารีที่ต้องติดตั้ง │ ├── schema.sql คำสั่งสร้างตารางฐานข้อมูล (รันครั้งเดียว) │ └── .env รหัสลับทั้งหมด (ไม่อัปโหลดขึ้น Git) │ └── frontend/ ← "หน้าตา" เว็บ (React) ├── index.html ├── vite.config.js ตั้งค่าตัวรันเว็บช่วงพัฒนา ├── package.json รายชื่อไลบรารีของฝั่งเว็บ └── src/ ├── main.jsx จุดเริ่มต้นของเว็บ ├── App.jsx กำหนดเมนู + หน้าเว็บทั้งหมด ├── api.js รวมฟังก์ชันเรียก backend ไว้ที่เดียว ├── styles.css สีและสไตล์ทั้งเว็บ ├── context/ ข้อมูลที่ใช้ร่วมกันทั้งเว็บ ├── hooks/ ฟังก์ชันช่วยดึงข้อมูลซ้ำเป็นระยะ ├── utils/ ฟังก์ชันช่วยเล็กๆน้อยๆ ├── pages/ หน้าเว็บหลักแต่ละหน้า └── components/ ชิ้นส่วน UI ย่อยที่แต่ละหน้าเอาไปประกอบกัน

03แต่ละไฟล์ทำหน้าที่อะไร

รายละเอียดของไฟล์สำคัญ ๆ ที่ถ้าต้องแก้อะไรในอนาคต จะได้รู้ว่าไปแก้ที่ไหน

ฝั่ง backend (สมอง)

ไฟล์หน้าที่
main.pyทุก endpoint (ช่องทางรับ-ส่งข้อมูล) ของเซิร์ฟเวอร์ รวมถึงตัวเชื่อม Supabase และ Gemini AI
schema.sqlคำสั่งสร้างตาราง weather_readings (ค่าเซนเซอร์), weather_predictions (ค่าทำนาย) และฟังก์ชันสรุปสถิติ — รันใน Supabase ครั้งเดียวตอนตั้งโปรเจกต์ใหม่
requirements.txtรายชื่อไลบรารี Python เช่น fastapi, supabase, google-genai

ช่องทางรับ-ส่งข้อมูล (endpoint) ที่มีอยู่ตอนนี้

Endpointใช้ทำอะไร
GET /healthเช็คว่าเซิร์ฟเวอร์และฐานข้อมูลยังทำงานปกติไหม
GET /weatherค่าอากาศล่าสุดตอนนี้
GET /historyค่าอากาศย้อนหลัง 1 ชั่วโมง (ใช้วาดกราฟหน้าแรก)
GET /readings-logค่าอากาศดิบทั้งหมด แบ่งหน้า + กรองตามช่วงเวลาได้
GET /predictตารางพยากรณ์ล่วงหน้า (ใช้ผลจาก AI จริงถ้ามี ไม่งั้นประมาณเอง)
GET /predictions-logเทียบค่าที่ทำนายไว้ กับค่าจริงที่เกิดขึ้นภายหลัง
GET /statsสรุปสถิติย้อนหลัง (เฉลี่ย/ต่ำสุด/สูงสุด) ให้ AI ตอบคำถามเชิงประวัติ
POST /ingestช่องทางให้ Node-RED ส่งค่าเซนเซอร์ใหม่เข้ามา (ต้องมีรหัสผ่านแนบมาด้วย)
POST /ingest-predictionช่องทางให้โมเดล AI ส่งผลทำนายเข้ามา
POST /askถาม-ตอบกับผู้ช่วย AI ในแชท

ฝั่ง frontend (หน้าตา)

ไฟล์ / โฟลเดอร์หน้าที่
App.jsxกำหนดเมนูด้านซ้าย และเส้นทาง (URL) ของแต่ละหน้า
api.jsรวมฟังก์ชันเรียก backend ทั้งหมด — ถ้า backend เพิ่ม endpoint ใหม่ ต้องมาเพิ่มที่นี่
context/LiveData.jsxดึงข้อมูล /health และ /weather ครั้งเดียว แล้วแจกจ่ายให้ทุกหน้าใช้ร่วมกัน (กันยิงซ้ำ)
pages/OverviewPage.jsxหน้าแรก — การ์ดตัวเลข + กราฟแนวโน้ม 1 ชม.
pages/ForecastPage.jsxตารางพยากรณ์ล่วงหน้า
pages/TrendComparePage.jsxกราฟเทียบ "ค่าจริง" กับ "ค่าทำนาย" แยกทีละพารามิเตอร์
pages/HistoryPage.jsxประวัติข้อมูล มี 2 แท็บ: ข้อมูลดิบ / ข้อมูลเทียบ
pages/SettingsPage.jsxตั้งค่าธีมเว็บ (dark/light)
components/ChatWidget.jsxปุ่มแชท AI มุมล่างขวา รวมปุ่มเลือกเสียง/ปิดเสียง
components/SensorAlert.jsxแจ้งเตือนเมื่อไม่มีข้อมูลเซนเซอร์ใหม่เกิน 10 นาที
components/ConnectionAlert.jsxแจ้งเตือนเมื่อเว็บต่อเซิร์ฟเวอร์ไม่ได้เลย
มีไฟล์เก่าเหลืออยู่ 3 ไฟล์ ChatBox.jsx, HistoryTable.jsx, WeatherCards.jsx ใน components/ เป็นไฟล์จากเวอร์ชันก่อนปรับปรุง ไม่ได้ถูกใช้งานจริงแล้ว — เห็นในโฟลเดอร์แต่ไม่ต้องสนใจ

04.env คืออะไร อยู่ไหน

ไฟล์เก็บ "รหัสลับ" ที่ห้ามให้ใครเห็น เช่น รหัสผ่านฐานข้อมูล และคีย์ AI

ไฟล์นี้อยู่ที่ backend/.env เพียงจุดเดียวในโปรเจกต์ (ฝั่งเว็บไม่มีไฟล์ .env ของตัวเอง)

ห้ามอัปโหลดไฟล์นี้ขึ้น GitHub เด็ดขาด ไฟล์ .gitignore ถูกตั้งไว้ให้ข้ามไฟล์ .env โดยอัตโนมัติอยู่แล้ว — แค่อย่าไปแก้ .gitignore ทิ้งบรรทัดนั้น และอย่า copy ค่าในไฟล์นี้ไปวางในแชทหรือที่สาธารณะ

ข้างในมีอะไรบ้าง

SUPABASE_URL
ที่อยู่ของฐานข้อมูล — หาได้จากเว็บ Supabase → โปรเจกต์ → Settings → API
SUPABASE_SERVICE_KEY
รหัสผ่านเข้าฐานข้อมูลแบบเต็มสิทธิ์ — อยู่หน้าเดียวกับ SUPABASE_URL
INGEST_TOKEN
รหัสผ่านที่ Node-RED ต้องแนบมาด้วยเวลาส่งค่าเซนเซอร์เข้า — ตั้งเองเป็นข้อความสุ่มอะไรก็ได้ แต่ต้องตรงกับที่ตั้งไว้ใน Node-RED
GEMINI_API_KEY
คีย์เข้าใช้ AI (Gemini) — ขอได้ฟรีที่ Google AI Studio
GEMINI_MODEL
ชื่อรุ่นของ AI ที่ใช้ (ตอนนี้ตั้งเป็น gemini-3.1-flash-lite)

05คำสั่ง Git หลักๆ

Git คือระบบเก็บประวัติการแก้โค้ด ส่วน GitHub คือที่เก็บออนไลน์ — repo นี้อยู่ที่ github.com/fern2548/voice-ai สาขาหลักชื่อ main

git status
ดูว่าไฟล์ไหนถูกแก้ไปบ้าง ยังไม่ได้บันทึก — ใช้เช็คก่อนทำอะไรทุกครั้ง
git add .
เลือกไฟล์ที่แก้ทั้งหมด เตรียมจะบันทึก
git commit -m "..."
บันทึกการแก้ไข พร้อมข้อความอธิบายสั้นๆ ว่าแก้อะไร
git push
ส่งสิ่งที่บันทึกไว้ขึ้น GitHub ให้คนอื่นในทีมเห็นด้วย
git pull
ดึงงานล่าสุดที่คนอื่นเพิ่มเข้ามาใน GitHub มาไว้ในเครื่องตัวเอง
git log --oneline
ดูประวัติการบันทึกทั้งหมดแบบย่อ

ลำดับที่ใช้บ่อยที่สุด (บันทึกงานแล้วส่งขึ้น GitHub)

# 1. ดูว่าแก้อะไรไปบ้าง
git status

# 2. เลือกทุกไฟล์ที่แก้ไว้
git add .

# 3. บันทึกพร้อมข้อความอธิบาย
git commit -m "แก้หน้า Settings"

# 4. ส่งขึ้น GitHub
git push
ก่อนเริ่มทำงานทุกครั้ง รัน git pull ก่อนเสมอ กันพลาดไปแก้ทับงานคนอื่นในทีมที่เพิ่ง push เข้ามา

06วิธีรันโปรเจกต์

ต้องรัน 2 โปรแกรมแยกกัน (คนละหน้าต่าง terminal) เพราะ backend กับ frontend เป็นคนละโปรแกรม

ฝั่ง backend (สมอง)

เปิด terminal แล้วเข้าโฟลเดอร์ backend
ติดตั้งไลบรารีครั้งแรก (ครั้งเดียวพอ): pip install -r requirements.txt
รันเซิร์ฟเวอร์: python main.py — ระบบจะลองพอร์ต 8000, 8001, 8002 ให้อัตโนมัติ แล้วบอกในหน้าจอว่าใช้พอร์ตไหน

ฝั่ง frontend (หน้าตา)

เปิด terminal อีกหน้าต่าง แล้วเข้าโฟลเดอร์ frontend
ติดตั้งไลบรารีครั้งแรก: npm install
รันเว็บ: npm run dev แล้วเปิดเบราว์เซอร์ไปที่ http://localhost:5173
ถ้า backend ไม่ได้อยู่พอร์ต 8000 ให้รัน frontend ด้วยคำสั่ง VITE_BACKEND_PORT=8001 npm run dev แทน (เปลี่ยน 8001 เป็นพอร์ตจริงที่ backend บอกไว้)

07ศัพท์ที่ควรรู้

คำศัพท์เทคนิคที่เจอบ่อยในเอกสารนี้ อธิบายแบบไม่ต้องมีพื้นฐาน

Backend
โปรแกรมที่ทำงานเบื้องหลัง ไม่มีหน้าตาให้เห็น รับ-ประมวลผล-เก็บข้อมูล
Frontend
ส่วนที่คนเห็นและกดใช้งานได้ผ่านเบราว์เซอร์
Endpoint
ช่องทาง/ที่อยู่หนึ่งจุดที่ frontend ยิงคำขอไปหา backend เช่น /weather
Database (Supabase)
ที่เก็บข้อมูลถาวร เหมือนตู้เก็บเอกสารดิจิทัลที่ค้นหาได้เร็ว
API key
รหัสผ่านสำหรับให้โปรแกรมของเราคุยกับบริการภายนอกได้ (เช่น Google Gemini AI)
Repository (repo)
โฟลเดอร์โปรเจกต์ทั้งหมดที่ถูกเก็บประวัติไว้ด้วย Git
Deploy
การเอาโปรแกรมขึ้นไปรันบนเซิร์ฟเวอร์จริง ให้คนอื่นเข้าใช้งานได้ตลอดเวลา (ไม่ใช่แค่รันในเครื่องตัวเอง)

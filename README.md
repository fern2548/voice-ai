# Weather Station AI — สถานีตรวจอากาศอัจฉริยะ

ระบบแดชบอร์ดสำหรับดูข้อมูลอากาศจากเซนเซอร์ที่ฟาร์ม (อุณหภูมิ ความชื้น ลม ฝน แสง)
พร้อมตารางพยากรณ์ล่วงหน้า และผู้ช่วย AI ที่ถาม-ตอบเรื่องสภาพอากาศได้ (พิมพ์หรือพูดก็ได้)

ระบบแบ่งเป็น 2 ส่วนที่ทำงานคู่กัน เหมือนร้านอาหาร:

| ส่วน | เปรียบเหมือน | ทำหน้าที่อะไร |
|---|---|---|
| **backend** (โฟลเดอร์ `backend/`) | ครัว | รับข้อมูลเซนเซอร์เข้ามาเก็บ, คำนวณ/ดึงค่าพยากรณ์, คุยกับ AI (Gemini), เก็บข้อมูลลงฐานข้อมูล |
| **frontend** (โฟลเดอร์ `frontend/`) | หน้าร้าน/จอแสดงผล | หน้าเว็บที่เราเห็นบนจอ กราฟ ตาราง ปุ่มกด แชท |

สองส่วนนี้เป็นคนละโปรแกรม ต้อง**เปิดพร้อมกันทั้งคู่**ระบบถึงจะใช้งานได้ (เหมือนต้องมีทั้งครัวและหน้าร้านร้านถึงจะขายได้)

---

## เตรียมเครื่องก่อนใช้งานครั้งแรก

ต้องลงโปรแกรม 2 ตัวนี้ก่อน (ลงครั้งเดียว):

1. **Python** (สำหรับรันส่วนครัว/backend) — โหลดที่ [python.org](https://www.python.org/downloads/)
2. **Node.js** (สำหรับรันส่วนหน้าร้าน/frontend) — โหลดที่ [nodejs.org](https://nodejs.org/) (เลือกเวอร์ชัน LTS)

ระหว่างติดตั้ง Python บน Windows ให้ติ๊กช่อง **"Add Python to PATH"** ด้วย ไม่งั้นจะเรียกใช้ผ่าน terminal ไม่ได้

---

## ขั้นที่ 1: ตั้งค่าความลับ (Environment Variables)

ระบบต้องใช้ "กุญแจลับ" เชื่อมต่อกับฐานข้อมูลและ AI ซึ่งเป็นข้อมูลที่ **ห้ามแชร์ให้คนอื่นหรือ commit ขึ้น GitHub**

1. เข้าโฟลเดอร์ `backend/`
2. สร้างไฟล์ชื่อ `.env` (ไม่มีนามสกุลอื่นต่อท้าย) แล้วใส่ข้อความนี้ (แก้ค่าให้ตรงกับของจริง):

```
SUPABASE_URL=https://xxxxxxxx.supabase.co
SUPABASE_SERVICE_KEY=ใส่คีย์จาก Supabase ตรงนี้
INGEST_TOKEN=ตั้งรหัสผ่านเองก็ได้ เอาไว้กันคนอื่นยิงข้อมูลปลอมเข้ามา
GEMINI_API_KEY=ใส่คีย์ AI จาก Google AI Studio (ถ้ายังไม่มี ระบบจะตอบคำถามแบบง่าย ๆ แทนได้)
```

> **Supabase** คือฐานข้อมูลที่เก็บค่าที่เซนเซอร์วัดได้ (เหมือนสมุดบันทึกออนไลน์)
> **Gemini** คือ AI ของ Google ที่ใช้ตอบคำถามในแชท ถ้าไม่ใส่คีย์นี้ ระบบจะตอบแบบกฎง่าย ๆ แทน (เช่น "ตอนนี้อุณหภูมิ 32°C") ยังใช้งานได้แต่ไม่ฉลาดเท่า

---

## ขั้นที่ 2: เปิดส่วนครัว (backend)

เปิด terminal (บน Windows คือ PowerShell หรือ Command Prompt) พิมพ์ทีละบรรทัด:

```bash
cd backend
pip install -r requirements.txt
python main.py
```

ถ้าสำเร็จจะเห็นข้อความประมาณ `Uvicorn running on http://0.0.0.0:8000`
**อย่าปิดหน้าต่างนี้** ปล่อยให้มันรันค้างไว้ตลอดเวลาที่ใช้งาน

---

## ขั้นที่ 3: เปิดส่วนหน้าร้าน (frontend)

เปิด terminal **หน้าต่างใหม่อีกอัน** (อย่าไปพิมพ์ทับหน้าต่างของ backend) แล้วพิมพ์:

```bash
cd frontend
npm install
npm run dev
```

จะมีลิงก์ขึ้นมาประมาณ `http://localhost:5173` — กด Ctrl+คลิก หรือก็อปไปเปิดในเบราว์เซอร์
ก็จะเห็นหน้าแดชบอร์ดแล้ว 🎉

> `npm install` ต้องรันแค่ **ครั้งแรก** ครั้งเดียว (หรือเวลามีคนแก้ไฟล์ `package.json`) ครั้งต่อไปพิมพ์แค่ `npm run dev` พอ

---

## หน้าจอในระบบมีอะไรบ้าง

| เมนู | ใช้ดูอะไร |
|---|---|
| **Overview** | ค่าที่วัดได้ตอนนี้ (real-time) + กราฟย้อนหลัง 1 ชั่วโมง |
| **Forecast** | ตารางพยากรณ์อากาศล่วงหน้าเป็นรายชั่วโมง |
| **Predictive Insights** | กราฟเทียบ "ค่าที่พยากรณ์ไว้" กับ "ค่าจริงที่วัดได้" ว่าโมเดลทำนายแม่นแค่ไหน |
| **History** | ดูข้อมูลดิบย้อนหลังทั้งหมด (export เป็น CSV ได้) |
| **Settings** | ปรับตั้งค่าทั่วไปของแดชบอร์ด |

มุมล่างขวามีปุ่มแชท 🤖 กดเพื่อถามคำถามเกี่ยวกับสภาพอากาศได้ พิมพ์หรือกดไมค์พูดก็ได้

## ปัญหาที่เจอบ่อย

**"เปิดหน้าเว็บแล้วไม่มีข้อมูล / ขึ้นเตือนว่าเชื่อมต่อเซิร์ฟเวอร์ไม่ได้"**
→ เช็คว่าหน้าต่าง terminal ของ backend (ขั้นที่ 2) ยังเปิดค้างอยู่ไหม ถ้าปิดไปแล้วให้เปิดใหม่

**"รัน `python main.py` แล้วขึ้น error ว่า SUPABASE_URL"**
→ ยังไม่ได้สร้างไฟล์ `.env` ในโฟลเดอร์ `backend/` หรือใส่ค่าไม่ครบ (ดูขั้นที่ 1)

**"หน้าเว็บที่ localhost:5173 บอกว่าเชื่อมต่อไม่ได้"**
→ backend ต้องรันอยู่ที่พอร์ต 8000 เท่านั้น (ห้ามมีโปรแกรมอื่นแย่งพอร์ต 8000 ใช้อยู่พร้อมกัน)

## โครงสร้างไฟล์ (ไว้อ้างอิงเวลาต้องแก้โค้ด)

```
voice-ai/
├── backend/                     # ส่วนครัว (Python)
│   ├── main.py                  # สมองหลัก: รับ/ส่งข้อมูลทุกอย่าง
│   ├── requirements.txt         # รายชื่อโปรแกรมย่อยที่ backend ต้องใช้
│   ├── schema.sql                # โครงสร้างตารางในฐานข้อมูล
│   └── .env                     # ไฟล์กุญแจลับ (ต้องสร้างเอง ไม่อยู่ใน git)
│
└── frontend/                    # ส่วนหน้าร้าน (React)
    ├── index.html
    ├── vite.config.js           # ตั้งค่าให้หน้าเว็บคุยกับ backend พอร์ต 8000
    ├── package.json
    └── src/
        ├── main.jsx              # จุดเริ่มต้นของหน้าเว็บ
        ├── App.jsx                # กำหนดเมนู/หน้าต่าง ๆ ทั้งหมด
        ├── api.js                 # ฟังก์ชันเรียกข้อมูลจาก backend
        ├── context/LiveData.jsx   # ที่เก็บข้อมูล real-time ให้ทุกหน้าใช้ร่วมกัน
        ├── pages/                 # แต่ละไฟล์ = แต่ละเมนูในแดชบอร์ด
        └── components/            # ชิ้นส่วนย่อย ๆ เช่น ตาราง, กราฟ, ป้ายเตือน
```

**อยากแก้อะไร ไปที่ไฟล์ไหน:**
- อยากเปลี่ยนวิธีคำนวณพยากรณ์ → `backend/main.py` ฟังก์ชัน `predict()`
- อยากเปลี่ยนคำตอบของ AI แชท → `backend/main.py` ฟังก์ชัน `ask()`
- อยากเปลี่ยนหน้าตา/สี/ฟอนต์ → `frontend/src/styles.css`
- อยากเพิ่มเมนูใหม่ → เพิ่มไฟล์ใน `frontend/src/pages/` แล้วไปเชื่อมที่ `frontend/src/App.jsx`
